GOAL
Extend user creation in the Events platform to support a strongly-typed EventsSignupContext object, while setting signupSource to "EVENTS_APP" for all Events-originated users.

BACKGROUND
Users created via the Events application are currently stored with:
signupSource = "API"

We already agreed to change this to:
signupSource = "EVENTS_APP"

We now want to formalize and enforce a structured signupContext specifically for Events using a typed interface, so event-related metadata is consistent, queryable, and future-proof.

CHANGE REQUEST
1. Ensure all users created via the Events application explicitly set:
   signupSource = "EVENTS_APP"

2. Add support for a typed signupContext object that follows this interface:

   interface EventsSignupContext {
     eventId: string;
     eventType: "METHOD" | "SKILL_SCHOOL" | "DIAMOND_CAMP";
     market: string;
     role: "attendee" | "guest" | "admin_added";
   }

3. signupContext should be optional, nullable, and stored as a JSON / object field.
   - It should only be populated for Events-created users.
   - Other user creation flows should remain unchanged.
